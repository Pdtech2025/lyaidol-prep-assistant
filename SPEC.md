# 云平台开发前期准备助手 — 完整规格文档

> **版本**: v2.0 | 2026-05-30
> **项目名**: lyaidol-prep-assistant
> **技术栈**: Next.js 16 (App Router) + React 19 + TypeScript 5 + Tailwind CSS 4 + shadcn/ui
> **数据来源**: Lyaidol B端云平台开发规范文档 + B端功能需求与开发计划Excel

---

## 一、产品定位

面向开发者的 **智能文档分析与准备工作台**，帮助开发团队在启动 Lyaidol B端云平台开发前，系统性地理解文档、提取重点、规划任务、答疑解惑，并集成扣子大模型的 TTS/ASR/音色声纹定制能力。同时整合Excel功能需求清单，提供14模块全景矩阵、智能idol专项准备、API确认追踪等专项能力。

### 1.1 核心价值

| 能力 | 说明 |
|------|------|
| 文档解析 | 输入URL或粘贴文本 → 自动抓取解析 → 结构化渲染 |
| 智能摘要 | LLM流式输出项目架构、数据模型关系、API分组统计、待确认项清单 |
| 准备清单 | LLM生成结构化开发前期任务清单，支持勾选进度跟踪 |
| AI问答 | 携带文档上下文的多轮对话，流式输出针对性建议 |
| TTS/ASR | 语音播报回复、语音输入提问、任务清单语音朗读 |
| 音色管理 | 预设音色选择 + 自定义音色录入（扣子平台复刻后speaker_id）+ 试听预览 |
| 模块矩阵 | 14个功能模块全景图 + Excel需求合并 + 模块级AI分析 |
| idol专项 | 30项智能idol配置准备 + 关联体模型 + 形象状态机 + 开发依赖梳理 |
| API追踪 | 105接口确认进度追踪 + AI推测⏳接口参数 + 批量导出 |

### 1.2 数据来源

本应用的分析对象为 **Lyaidol B端云平台** 的完整开发规范文档，核心信息摘要如下：

- **项目**: lyaidol-platform（Vue3 + Element Plus + TypeScript）
- **定位**: 二次元偶像周边商家的后台管理系统，与C端微信小程序共享同一后端
- **后端Base URL**: `https://lyaidol.leyard.com`
- **认证**: `POST /merchant/auth/login` → `Authorization: Bearer {token}`
- **API总数**: 105个（27个✅已确认 + 78个⏳待后端确认）
- **数据模型**: 7大实体组（用户与权限、会员、商品与订单、设备、AI/Aidol、应援、反馈）
- **页面**: 17个页面，分5个开发阶段（P1-P5）
- **侧边栏**: 首页 + 商品管理 + 订单管理 + 设备管理 + 智能Aidol + UI风格 + 会员管理 + 应援管理 + 权限管理 + 意见反馈

**#1 抓取/粘贴文档**

```
POST /api/fetch-url
请求: {
  url?: string,          // 方式A: 输入URL抓取
  content?: string,      // 方式B: 直接粘贴文本
  title?: string         // 可选标题
}
处理:
  - 若提供url → FetchClient抓取解析 → 自动存S3 → 返回docId
  - 若提供content → 直接存S3 → 返回docId
响应: {
  docId: string,         // 文档唯一ID，后续接口引用
  title: string,         // 文档标题
  content: string,       // 结构化文档内容
  stats: {               // 文档统计
    totalLines: number,
    totalChars: number,
    modules: string[],   // 识别到的模块列表
    apiCount: number,    // API接口数量
    dataModels: number   // 数据模型数量
  }
}
```

## 五、音色声纹定制方案

### 5.1 现状分析

| 调查项 | 结论 |
|--------|------|
| `coze-coding-dev-sdk` 是否有音色复刻API | 没有。SDK只有TTSClient和ASRClient |
| Coze OpenAPI 是否有音色训练端点 | 没有。探测了14个路径均返回404 |
| TTSClient 是否支持复刻音色 | 支持。speaker参数接受任意字符串 |
| 扣子TTS插件 | 支持"系统预设音色和资源库中复刻出的音色" |
| 复刻功能操作路径 | 扣子Web平台 → 资源库 → 音色复刻 |

### 5.2 双轨制方案

| 轨道 | 方式 | 说明 |
|------|------|------|
| A. 扣子平台复刻 | 用户在扣子Web完成音色复刻 → 获得speaker_id → 在音色管理页录入 | 最稳定可靠 |
| B. 本地音色库管理 | 用户上传音频样本 → 存S3 → 管理元数据 → 关联speaker_id | 本地管理资产 |

### 5.3 数据存储

| 数据 | 存储位置 | 路径 |
|------|---------|------|
| 自定义音色元数据 | S3Storage | `voice-library/{voiceId}.json` |
| 样本音频文件 | S3Storage | `voice-samples/{voiceId}.{ext}` |
| 预设音色列表 | 前端硬编码 | 15+个固定音色 |
| 试听音频 | 临时生成 | TTSClient实时合成，不存储 |

## 八、设计风格方向

| 维度 | 规范 |
|------|------|
| **气质意象** | 专业开发者工作台 — 深色主题，安静高效，不喧哗 |
| **配色** | 暗色底 `#0f1117` + 青蓝主色 `#3b82f6 → #06b6d4` 渐变，信息密度优先 |
| **字体** | 思源黑体 + JetBrains Mono（代码区域） |
| **布局** | 左右分栏，左侧固定导航（可收起），右侧主内容区 |
| **交互** | 流式打字机效果、任务勾选即时反馈、文档解析进度动画、语音播报波形 |
| **设计禁忌** | 不用科技蓝+圆角卡片万能组合；不用浅色主题默认；不用大留白低信息密度 |

### 10.5 业务规则

**状态机**:
- 订单: 待支付(0)→已支付(1)→已完成; 待支付→已取消(2); 已支付→已退款(3)
- 设备: 未激活(0)→已激活(1); 未绑定(0)→已绑定(1)[C端触发]; 离线(0)↔在线(1)[设备上报]
- 抽奖: 未开始(0)→进行中(1)→已结束(2)
- 消息: 待发送(0)→已发送(1)→已过期(2)

**权限**: 超级管理员(全部权限不可编辑) → 自定义角色(菜单权限树) → 粒度: 菜单可见→操作按钮→数据范围

**校验**: 手机号11位1开头 | 邮箱标准格式 | 商品名max100 | 角色名max50 | 分类名max20 | 消息max2000 | 密码min8含大小写+数字+特殊字符

**C端关联**: 角色status=1→C端首页 | 轮播图status=1→C端首页 | 商品status=1→C端商城 | 消息status=1且到达sendTime→C端通知 | 抽奖status=1且在活动时间内→C端可参与
